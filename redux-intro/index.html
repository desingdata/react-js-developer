01 · Concepto

¿Qué es Redux?

Redux es una biblioteca de gestión de estado predecible para aplicaciones JS, inspirada en la arquitectura Flux de Facebook y la programación funcional.

🏛️

Single Source of Truth

Todo el estado de la aplicación se almacena en un único árbol de objetos dentro de un solo Store. Esto simplifica la depuración y el mantenimiento del estado global.

🔒

Estado de Solo Lectura

El estado nunca se modifica directamente. La única manera de cambiar el estado es emitiendo una acción (action), un objeto que describe qué ocurrió.

⚡

Cambios con Funciones Puras

Los Reducers son funciones puras: dado el mismo estado y la misma acción, siempre devuelven el mismo resultado, sin efectos secundarios.

🔄

Flujo Unidireccional

Los datos fluyen en una sola dirección: Action → Reducer → Store → View. Esta predictibilidad reduce los bugs y facilita el testing.

03 · Integración

Comunicación con React

La librería react-redux conecta el Store con los componentes. Usa el Provider, useSelector y useDispatch.

index.jsx 
import { Provider } from 'react-redux';
import { store } from './store';

// PROVIDER: envuelve toda la app para que cualquier
// componente pueda acceder al Store de Redux.
ReactDOM.render(
  <Provider store={store}>
    <App />
  </Provider>,
  document.getElementById('root')
);
components/Counter.jsx 
import { useSelector, useDispatch } from 'react-redux';
import { increment, decrement } from './counterSlice';

function Counter() {
  // useSelector: suscribe el componente a una parte del Store.
  // Se re-renderiza sólo cuando ese trozo de estado cambia.
  const count = useSelector((state) => state.counter.value);

  // useDispatch: retorna la función dispatch para enviar acciones.
  const dispatch = useDispatch();

  return (
    <div>
      <span>{count}</span>
      <button onClick={() => dispatch(increment())}>+</button>
      <button onClick={() => dispatch(decrement())}>-</button>
    </div>
  );
}

04 · Componentes

Componentes Principales

Redux se construye sobre cuatro pilares fundamentales que trabajan en conjunto para gestionar el estado de forma predecible.

01

Store

El contenedor central que almacena el árbol de estado completo de la aplicación. Solo existe un Store por aplicación. Expone getState(), dispatch() y subscribe().

createStore() / configureStore()
02

Actions

Objetos planos de JavaScript que describen qué ocurrió en la aplicación. El campo type es obligatorio. Los Action Creators son funciones que los generan.

{ type: 'INCREMENT', payload: 1 }
03

Reducers

Funciones puras con la firma (state, action) => newState. Especifican cómo el estado de la aplicación cambia en respuesta a las acciones. Nunca mutan el estado directamente.

createReducer() / createSlice()
04

Selectors

Funciones que extraen y derivan datos del estado del Store. Con createSelector (Reselect) se memorizan para evitar recálculos innecesarios y mejorar el rendimiento.

createSelector()
features/counterSlice.js — Redux Toolkit 
import { createSlice } from '@reduxjs/toolkit';

// createSlice: genera el reducer, las actions y los action creators
// de forma automática, reduciendo el boilerplate de Redux clásico.
const counterSlice = createSlice({
  name: 'counter',

  // INITIAL STATE: valor por defecto del estado de este slice
  initialState: { value: 0, status: 'idle' },

  // REDUCERS: cada método aquí es un mini-reducer + su action creator.
  // Immer (incluido en RTK) permite "mutar" el draft de estado de forma segura.
  reducers: {
    increment: (state) => { state.value += 1; },
    decrement: (state) => { state.value -= 1; },
    incrementByAmount: (state, action) => {
      // action.payload contiene el valor enviado en el dispatch
      state.value += action.payload;
    },
  },
});

// Exportar los action creators generados automáticamente
export const { increment, decrement, incrementByAmount } = counterSlice.actions;

// SELECTOR: función que extrae el valor del contador del Store
export const selectCount = (state) => state.counter.value;

export default counterSlice.reducer;
