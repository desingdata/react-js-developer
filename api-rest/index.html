Proyecto práctico con Async/Await, Promises, Fetch y Axios en JavaScript

async Async / Await

Sintaxis de ES2017 que permite escribir código asíncrono con apariencia síncrona. async declara una función que siempre retorna una Promise; await pausa la ejecución hasta que la Promise se resuelva.
JavaScript — Async/Await 
// async marca que la función es asíncrona → retorna Promise implícitamente
async function obtenerUsuario(id) {
  try {
    // await "espera" la respuesta sin bloquear el hilo principal
    const respuesta = await fetch(`https://jsonplaceholder.typicode.com/users/${id}`);

    // Verificar si la solicitud HTTP fue exitosa (status 200-299)
    if (!respuesta.ok) throw new Error(`HTTP ${respuesta.status}`);

    // .json() también es asíncrono → necesita await para leer el body
    const datos = await respuesta.json();
    return datos;

  } catch (error) {
    // try/catch captura tanto errores de red como de lógica
    console.error('Error al obtener usuario:', error.message);
    throw error; // relanzar para manejo externo
  }
}

// Llamada con await dentro de otra función async
async function main() {
  const usuario = await obtenerUsuario(1);
  console.log(usuario.name, usuario.email);
}
main();

async Promise.all + Async/Await

Promise.all() ejecuta múltiples Promises en paralelo y espera a que todas finalicen. Combinado con await es ideal para peticiones concurrentes. 
async function obtenerVariosUsuarios() {
  // Las tres peticiones se lanzan en paralelo (no secuencialmente)
  const [u1, u2, u3] = await Promise.all([
    fetch('https://jsonplaceholder.typicode.com/users/1').then(r => r.json()),
    fetch('https://jsonplaceholder.typicode.com/users/2').then(r => r.json()),
    fetch('https://jsonplaceholder.typicode.com/users/3').then(r => r.json()),
  ]);
  // Si cualquiera falla → Promise.all rechaza toda la operación
  return [u1, u2, u3];
}

Promise Promises con Fetch

Una Promise representa un valor que puede estar disponible ahora, en el futuro o nunca. Tiene tres estados: pending, fulfilled y rejected. fetch() es la API nativa del navegador que retorna una Promise para peticiones HTTP. 
// fetch() retorna una Promise de tipo Response
fetch('https://jsonplaceholder.typicode.com/posts/1')

  // .then() maneja el caso exitoso → recibe la Response
  .then((respuesta) => {
    if (!respuesta.ok) {
      // Lanzar error manual para que lo capture .catch()
      throw new Error(`Código HTTP: ${respuesta.status}`);
    }
    // .json() retorna OTRA Promise → encadenamos otro .then()
    return respuesta.json();
  })

  // Segundo .then() recibe los datos ya parseados como objeto JS
  .then((datos) => {
    console.log('Título:', datos.title);
    console.log('Cuerpo:', datos.body);
  })

  // .catch() captura cualquier error de la cadena .then()
  .catch((error) => {
    console.error('Fallo en la petición:', error.message);
  })

  // .finally() se ejecuta siempre, haya o no error (ej: ocultar loading)
  .finally(() => console.log('Petición finalizada'));

Promise POST con Fetch

Para peticiones POST, PUT o DELETE se debe configurar el objeto options de fetch con el método, cabeceras y cuerpo serializado. 
// Enviar datos JSON con POST → se necesita serializar con JSON.stringify
fetch('https://jsonplaceholder.typicode.com/posts', {
  method: 'POST',                       // Método HTTP
  headers: {
    'Content-Type': 'application/json',  // Tipo del cuerpo
    'Authorization': 'Bearer mi-token'  // Token de autenticación
  },
  body: JSON.stringify({               // Convertir objeto a string JSON
    title: 'Nuevo post',
    body: 'Contenido del post',
    userId: 1
  })
})
  .then(r => r.json())
  .then(data => console.log('Creado con ID:', data.id));

axios Axios — Librería HTTP

Axios es una librería basada en Promises para realizar peticiones HTTP tanto en navegador como en Node.js. Simplifica el manejo de cabeceras, serialización automática de JSON, interceptores y cancelación de peticiones. 
// Instalación: npm install axios  |  CDN: <script src="https://cdn.jsdelivr.net/npm/axios">
import axios from 'axios';

// GET → axios parsea JSON automáticamente (sin .json())
axios.get('https://jsonplaceholder.typicode.com/users')
  .then(respuesta => {
    // La data viene directo en respuesta.data (sin paso extra)
    console.log(respuesta.data);
    console.log('Status:', respuesta.status);
    console.log('Headers:', respuesta.headers);
  })
  .catch(error => {
    // Axios rechaza la Promise automáticamente en errores HTTP (4xx/5xx)
    if (error.response) {
      // El servidor respondió con un código de error
      console.error(error.response.data);
    } else if (error.request) {
      // No hubo respuesta del servidor (problema de red)
      console.error('Sin respuesta del servidor');
    }
  });

// POST → Axios serializa el cuerpo automáticamente como JSON
axios.post('/api/usuarios', { nombre: 'Ana', email: 'ana@test.com' })
  .then(res => console.log('Creado:', res.data.id));

axios Instancia personalizada + Interceptores

Una instancia de Axios permite configurar valores base (URL, headers, timeout) reutilizables. Los interceptores son funciones que se ejecutan antes de cada solicitud o después de cada respuesta, útiles para tokens y manejo global de errores. 
// Crear instancia con configuración base reutilizable
const apiClient = axios.create({
  baseURL: 'https://api.miapp.com/v1',  // URL base para todas las peticiones
  timeout: 5000,                       // Cancelar si no responde en 5s
  headers: { 'Accept': 'application/json' }
});

// Interceptor de solicitud → se ejecuta ANTES de enviar
apiClient.interceptors.request.use((config) => {
  const token = localStorage.getItem('token');
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;  // Siempre retornar config modificada
});

// Interceptor de respuesta → manejo global de errores
apiClient.interceptors.response.use(
  (respuesta) => respuesta,  // Caso exitoso: pasar sin cambios
  (error) => {              // Caso error: manejar globalmente
    if (error.response?.status === 401) {
      console.warn('Sesión expirada → redirigir al login');
    }
    return Promise.reject(error);
  }
);

// Uso de la instancia → la baseURL se agrega automáticamente
apiClient.get('/usuarios').then(r => console.log(r.data));

React Axios en Componentes React

En React, las peticiones HTTP se realizan dentro del hook useEffect que se ejecuta al montar el componente. El estado de carga, error y datos se gestiona con useState para controlar la UI reactivamente. 
import { useState, useEffect } from 'react';
import axios from 'axios';

// Hook personalizado → encapsula la lógica de fetch y es reutilizable
function useUsuarios() {
  const [datos, setDatos] = useState([]);      // Estado de los datos
  const [cargando, setCargando] = useState(true); // Estado de carga
  const [error, setError] = useState(null);       // Estado de error

  useEffect(() => {
    // AbortController → cancelar la petición si el componente se desmonta
    const controller = new AbortController();

    const cargar = async () => {
      try {
        setCargando(true);
        const { data } = await axios.get(
          'https://jsonplaceholder.typicode.com/users',
          { signal: controller.signal }  // Vincular señal de cancelación
        );
        setDatos(data);
      } catch (err) {
        if (!axios.isCancel(err)) setError(err.message); // Ignorar cancelaciones
      } finally {
        setCargando(false); // Siempre quitar el estado de carga
      }
    };

    cargar();
    // Cleanup: cancelar petición al desmontar el componente
    return () => controller.abort();
  }, []); // [] = solo se ejecuta al montar el componente

  return { datos, cargando, error };
}

// Componente que consume el hook personalizado
function ListaUsuarios() {
  const { datos, cargando, error } = useUsuarios();

  if (cargando) return <p>Cargando...</p>;
  if (error)    return <p>Error: {error}</p>;

  return (
    <ul>
      {datos.map(u => (
        <li key={u.id}>{u.name} — {u.email}</li>
      ))}
    </ul>
  );
}

📊 Comparativa: Fetch vs Axios

Ambas herramientas permiten consumir APIs REST, pero difieren en características, verbosidad y soporte del entorno. La elección depende del proyecto y las necesidades.
Característica Fetch (nativo) Axios (librería)
Instalación requerida ✓ No (nativo) ✗ npm install axios
Parseo JSON automático ✗ .json() manual ✓ response.data
Interceptores ✗ No incluido ✓ request/response
Timeout configurable ✗ Requiere AbortController ✓ opción timeout
Error en 4xx/5xx ✗ No rechaza (chequeo manual) ✓ Rechaza automáticamente
Node.js compatible ⚠️ Node 18+ nativo ✓ Todas las versiones
Instancias reutilizables ✗ No ✓ axios.create()
Peso bundle ✓ 0 kb ~14 kb (gzip)

⚡ Cuándo usar Async/Await

Cuando el código asíncrono es complejo o secuencial y necesitas claridad de lectura.
  • Múltiples pasos dependientes entre sí
  • Manejo de errores con try/catch
  • Flujos legibles tipo síncrono
  • Integración con for...of y loops

🔗 Cuándo usar Promise chain

Para transformaciones simples encadenadas donde la legibilidad tipo pipeline es preferida.
  • Transformaciones de datos en cadena
  • Uso de Promise.all / Promise.race
  • Composición funcional de operaciones
  • Callbacks simples sin mucha lógica
