Introducción a React Router

La librería estándar para navegación en aplicaciones React

1
¿Qué es React Router?

React Router es una librería de enrutamiento del lado del cliente para React. Permite crear aplicaciones de una sola página (SPA) donde la navegación ocurre sin recargar el navegador, mapeando las rutas de la URL a componentes específicos.

Single Page Application (SPA): Solo se carga el HTML inicial una vez. Los cambios de "página" son renderizados por JavaScript en el cliente, haciendo la navegación instantánea.
2
Instalación 
# Instalar React Router v6 en tu proyecto
npm install react-router-dom

# O con yarn
yarn add react-router-dom
Nota: react-router-dom incluye todo lo necesario para aplicaciones web. Para React Native se usa react-router-native.
3
Configuración inicial — main.jsx

El punto de entrada de la aplicación envuelve todo con el router principal.

// main.jsx — Punto de entrada de la aplicación
import { StrictMode } from 'react'
import { createRoot } from 'react-dom/client'

// BrowserRouter: habilita el enrutamiento en toda la app
import { BrowserRouter } from 'react-router-dom'
import App from './App'

createRoot(document.getElementById('root')).render(
  <StrictMode>
    {/* BrowserRouter envuelve toda la app para proveer contexto de routing */}
    <BrowserRouter>
      <App />
    </BrowserRouter>
  </StrictMode>
)

Componentes Principales

Los building blocks esenciales de React Router v6

1
BrowserRouter

Provee el contexto de enrutamiento a toda la aplicación usando la History API del navegador. Debe envolver el árbol de componentes donde se use el router.

import { BrowserRouter } from 'react-router-dom'

// BrowserRouter: usa URLs limpias tipo /about, /users/5
// (requiere configuración del servidor en producción)
function main() {
  root.render(
    <BrowserRouter>  {/* ← Contexto global */}
      <App />
    </BrowserRouter>
  )
}
Variantes de Router

BrowserRouter

URLs: /home, /about
Usa History API

HashRouter

URLs: /#/home
Para hosting estático

2
Routes y Route

<Routes> es el contenedor de todas las rutas. <Route> define qué componente renderizar para cada URL. En v6, solo se renderiza la primera ruta que coincide.

import { Routes, Route } from 'react-router-dom'
import { Home, About, User, NotFound } from './pages'

function App() {
  return (
    <>
      <Navbar />  {/* Siempre visible */}

      {/* Routes: busca la primera ruta que coincida con la URL */}
      <Routes>

        {/* Route: path → componente a renderizar */}
        <Route path="/"         element={<Home />} />
        <Route path="/about"    element={<About />} />

        {/* :id es un parámetro dinámico */}
        <Route path="/user/:id" element={<User />} />

        {/* * captura cualquier ruta no definida → 404 */}
        <Route path="*"          element={<NotFound />} />

      </Routes>
    </>
  )
}
Tabla de rutas
Path Componente Tipo
/ Home estática
/about About estática
/user/:id User dinámica
* NotFound fallback
3
Link y NavLink

<Link> reemplaza a <a href> para navegar sin recargar. <NavLink> añade estilos automáticamente a la ruta activa.

import { Link, NavLink } from 'react-router-dom'

function Navbar() {
  return (
    <nav>
      {/* Link: navegación básica sin recarga */}
      <Link to="/">Inicio</Link>

      {/* NavLink: aplica clase/estilo cuando la ruta está activa */}
      <NavLink
        to="/about"
        {/* Función: recibe {isActive} y retorna clase CSS */}
        className={({ isActive }) =>
          isActive ? "link activo" : "link"
        }
      >
        Acerca de
      </NavLink>

      {/* NavLink con estilos inline condicionales */}
      <NavLink
        to="/contacto"
        style={({ isActive }) => ({
          color: isActive ? '#61dafb' : '#8892b0',
          fontWeight: isActive ? 'bold' : 'normal'
        })}
      >
        Contacto
      </NavLink>
    </nav>
  )
}
Link vs <a>: Usar <a href> recarga toda la página. <Link> intercepta el click, actualiza la URL y solo re-renderiza el componente correcto.

Acceso a Ruta Actual

Hooks para leer y reaccionar a la URL activa

1
useLocation — Leer la URL completa

useLocation() retorna el objeto de ubicación actual con pathname, search (query params) y hash.

🔒 localhost:3000/productos?categoria=ropa#ofertas
import { useLocation } from 'react-router-dom'

function Breadcrumb() {
  // useLocation: retorna la ubicación actual del navegador
  const location = useLocation()

  // location.pathname → "/productos"
  // location.search   → "?categoria=ropa"
  // location.hash     → "#ofertas"
  // location.state    → datos pasados por navigate()

  return (
    <div>
      <p>Ruta actual: {location.pathname}</p>
      <p>Filtro: {location.search}</p>
    </div>
  )
}
Propiedades de location
Propiedad Ejemplo Uso
pathname /productos Ruta actual
search ?cat=ropa Query params
hash #ofertas Ancla de página
state { from: '/' } Estado de navegación
2
useParams — Parámetros dinámicos

useParams() extrae los parámetros definidos en el path de la ruta (los segmentos con :).

🔒 localhost:3000/user/42/post/javascript
import { useParams } from 'react-router-dom'

// La ruta está definida como: /user/:userId/post/:slug
function UserPost() {
  // useParams: obtiene los segmentos dinámicos de la URL
  const { userId, slug } = useParams()

  // Para /user/42/post/javascript:
  // userId → "42"  (siempre string)
  // slug   → "javascript"

  return (
    <div>
      <h1>Post: {slug}</h1>
      <p>Autor ID: {userId}</p>
    </div>
  )
}
Importante: Los parámetros de useParams() son siempre strings. Si necesitas un número, convierte con Number(userId) o parseInt(userId).
3
useMatch — Comparar ruta activa

useMatch() verifica si la URL actual coincide con un patrón dado. Útil para lógica condicional basada en la ruta.

import { useMatch } from 'react-router-dom'

function Sidebar() {
  // useMatch: retorna objeto si hay coincidencia, o null si no
  const enHome   = useMatch("/")
  const enPerfil = useMatch("/user/:id")

  return (
    <aside>
      {/* Mostrar banner solo en la home */}
      {enHome && <Banner />}

      {/* Mostrar el ID del perfil si estamos en /user/:id */}
      {enPerfil && (
        <p>Perfil: {enPerfil.params.id}</p>
      )}
    </aside>
  )
}

Redirecciones

Navegar y redirigir programáticamente entre rutas

1
Componente <Navigate>

<Navigate> redirige declarativamente cuando se renderiza. Ideal para redirecciones en el JSX.

import { Navigate } from 'react-router-dom'

// Navigate: redirige al renderizarse (declarativo)
function PaginaAntigua() {
  // replace: reemplaza la entrada del historial
  // (el usuario no puede volver con "atrás")
  return <Navigate to="/nueva-ruta" replace />
}

// Redirigir según condición (guard de autenticación)
function Dashboard() {
  const usuario = useAuth()  // hook personalizado

  // Si no está autenticado, redirigir al login
  if (!usuario) {
    return <Navigate to="/login" state={{ from: "/dashboard" }} />
  }

  return <div>Bienvenido {usuario.nombre}</div>
}
2
Hook useNavigate

useNavigate() retorna una función para navegar programáticamente. Ideal para redirecciones tras eventos (submit de formulario, click de botón).

import { useNavigate } from 'react-router-dom'

function FormularioLogin() {
  // useNavigate: retorna función navigate(to, options)
  const navigate = useNavigate()

  const handleSubmit = async (e) => {
    e.preventDefault()

    const ok = await loginAPI(email, pass)

    if (ok) {
      // Navegar al dashboard tras login exitoso
      navigate("/dashboard")
    }
  }

  return (
    <button onClick={() => navigate(-1)}>
      ← Volver atrás
    </button>
    {/* navigate(-1) → equivale al botón "Atrás" del browser */}
    {/* navigate(1)  → equivale al botón "Adelante" */}
  )
}

3
Ruta Protegida (PrivateRoute)

Patrón para proteger rutas que requieren autenticación, combinando Navigate con lógica condicional.

import { Navigate, Outlet } from 'react-router-dom'

// PrivateRoute: componente que protege rutas privadas
// Outlet renderiza los componentes hijos de la ruta
function PrivateRoute({ autenticado }) {
  return autenticado
    ? <Outlet />              // ✅ Renderiza la ruta hija
    : <Navigate to="/login" /> // 🔒 Redirige al login
}

// Uso en App.jsx con rutas anidadas
function App() {
  const isAuth = useAuth()
  return (
    <Routes>
      <Route path="/login" element={<Login />} />

      {/* Rutas protegidas como hijos de PrivateRoute */}
      <Route element={<PrivateRoute autenticado={isAuth} />}>
        <Route path="/dashboard" element={<Dashboard />} />
        <Route path="/perfil"    element={<Perfil />} />
      </Route>
    </Routes>
  )
}
Flujo de autenticación: Usuario intenta acceder a /dashboard → PrivateRoute verifica isAuth → si es false, redirige a /login → tras login exitoso, navigate('/dashboard').
