01 · Fundamentos

Instalación y configuración

React es una librería de JavaScript para construir interfaces de usuario. Antes de empezar, necesitas tener Node.js instalado (versión 18 o superior). Node incluye npm, el gestor de paquetes que usaremos.

🟢

Node.js

Entorno de ejecución de JS fuera del navegador. Requerido para usar npm.

📦

npm / yarn

Gestores de paquetes para instalar dependencias del proyecto.

⚡

Vite

Bundler moderno y rápido, alternativa recomendada a Create React App.

bash — terminal 
# Verificar que Node.js está instalado correctamente
node --version    # Debe mostrar v18.x.x o superior
npm  --version    # Viene incluido con Node.js

# Instalar React Developer Tools (extensión para Chrome/Firefox)
# Buscar "React DevTools" en la tienda de extensiones
02 · Estructura

Creación de proyecto

Usamos Vite para crear la estructura inicial del proyecto. Vite genera automáticamente todos los archivos de configuración, carpetas y dependencias necesarias para empezar a trabajar con React.

1

Crear el proyecto con Vite

Ejecutar el comando en la carpeta donde quieres el proyecto.

2

Seleccionar React + JavaScript

El asistente interactivo pregunta el framework y variante.

3

Instalar dependencias

npm install descarga todos los paquetes del package.json.

4

Iniciar servidor de desarrollo

npm run dev abre el proyecto en http://localhost:5173

bash — crear proyecto 
# Crear un nuevo proyecto React con Vite (scaffolding automático)
npm create vite@latest mi-app-react -- --template react

# Entrar a la carpeta del proyecto recién creado
cd mi-app-react

# Instalar todas las dependencias listadas en package.json
npm install

# Iniciar el servidor de desarrollo con Hot Module Replacement (HMR)
npm run dev

03 · Componentes

Introducción a componentes

Los componentes son el núcleo de React. Son funciones de JavaScript que devuelven JSX (HTML con superpoderes). Cada componente es una pieza reutilizable e independiente de la interfaz.

jsx — componente funcional básico 
// Un componente es simplemente una función que retorna JSX.
// JSX = JavaScript XML: mezcla HTML con expresiones JS.

function TarjetaUsuario({ nombre, rol, avatar }) {
  // Las props son los parámetros del componente (datos de entrada).
  // Se pasan como atributos HTML desde el componente padre.

  return (
    // El JSX debe tener un solo elemento raíz (o usar Fragment <>)
    <div className="tarjeta">
      <img src={avatar} alt={nombre} />

      {"/* Las llaves {} ejecutan expresiones JS dentro del JSX */"}
      <h2>{nombre}</h2>
      <p>{rol}</p>
    </div>
  );
}

// Usar el componente (instanciar) pasándole sus props:
<TarjetaUsuario
  nombre="Ana García"
  rol="Frontend Dev"
  avatar="/foto.jpg"
/>
jsx — useState: estado del componente 
// useState es un Hook: función especial de React que permite
// guardar y actualizar datos dentro de un componente.

import { useState } from 'react';

function Contador() {
  // useState(0) → valor inicial
  // count → valor actual del estado
  // setCount → función para actualizar el estado (re-renderiza)
  const [count, setCount] = useState(0);

  return (
    <div>
      {"/* Mostrar el valor actual del estado */"}
      <p>Clics: {count}</p>

      {"/* onClick es el manejador de eventos (event handler) */"}
      <button onClick={() => setCount(count + 1)}>
        Incrementar
      </button>
    </div>
  );
}

// Regla fundamental: NUNCA mutar el estado directamente.
// ✘ count++ → incorrecto (no activa re-render)
// ✔ setCount(count + 1) → correcto (React actualiza la UI)
jsx — listas y renderizado condicional 
function ListaTareas({ tareas }) {
  return (
    <ul>
      {tareas.map((tarea) => (
        {"/* key: identificador único para optimización del Virtual DOM */"}
        <li key={tarea.id}>

          {"/* Renderizado condicional: muestra icono solo si está hecha */"}
          {tarea.hecha && <span>✅</span>}

          {tarea.texto}
        </li>
      ))}
    </ul>
  );
}

// Array de datos de ejemplo para la lista:
const misTareas = [
  { id: 1, texto: 'Aprender React',  hecha: true  },
  { id: 2, texto: 'Crear componentes', hecha: false },
  { id: 3, texto: 'Hacer deploy',      hecha: false },
];
04 · Mentalidad

Pensar en React

La guía oficial de React propone un proceso de 5 pasos para diseñar cualquier interfaz. El objetivo es descomponer el problema en piezas pequeñas y manejar los datos correctamente.

Paso 1

Dividir en componentes

Analiza el diseño/mockup y dibuja cajas alrededor de cada parte lógica de la UI.

Paso 2

Construir la versión estática

Implementa la UI con props únicamente. Sin estado todavía. Solo renderizado.

Paso 3

Identificar el estado mínimo

¿Qué datos cambian en el tiempo? Ese es tu estado. El resto son props o derivados.

Paso 4

¿Dónde vive el estado?

Ubica el estado en el ancestro común más cercano de todos los que lo necesitan.

Paso 5

Agregar flujo de datos inverso

Los hijos notifican cambios al padre pasando callbacks como props. Los datos fluyen hacia abajo (props), los eventos hacia arriba (callbacks).

jsx — flujo de datos: props ↓ · eventos ↑ 
// PADRE: dueño del estado, lo pasa hacia abajo con props.
function App() {
  const [busqueda, setBusqueda] = useState('');

  // setBusqueda se pasa al hijo como prop (callback)
  return (
    <>
      <BarraBusqueda onBuscar={setBusqueda} />
      <TablaProductos filtro={busqueda} />
    </>
  );
}

// HIJO A: recibe callback, notifica al padre los cambios de input.
function BarraBusqueda({ onBuscar }) {
  return (
    <input
      placeholder="Buscar..."
      {"/* onChange: evento del DOM → callback del padre */"}
      onChange={(e) => onBuscar(e.target.value)}
    />
  );
}

// HIJO B: recibe el filtro como prop y filtra los productos.
function TablaProductos({ filtro }) {
  const resultado = productos.filter(p =>
    p.nombre.toLowerCase().includes(filtro.toLowerCase())
  );
  return <ul>{resultado.map(p => <li key={p.id}>{p.nombre}</li>)}</ul>;
}
